# Flight-search spec

## Purpose

提供省域机票搜索和甩尾航班发现：用户按省份搜索最低票价航班，系统自动发现经停目标城市的低价联程航班。

## ADDED Requirements

### Requirement: 用户可按省份搜索航班

系统 SHALL 接受出发地城市、目标省份和出发日期，返回省内每个机场的最低直飞航班列表，按价格升序排列。

#### Scenario: 省域搜索成功

- **WHEN** 用户在 `/app/flights` 选择出发地"上海"、目的地"湖南省"、出发日期"2026-07-15"并点击搜索
- **THEN** 系统返回湖南省内各机场（长沙、张家界、常德等）的最低价格航班
- **AND** 结果按价格升序排列
- **AND** 每个航班包含：航司名称、航班号、出发时间、到达时间、价格

#### Scenario: 省份无机场

- **WHEN** 用户选择的目的地省份在静态数据中无机场记录
- **THEN** 系统显示"该省份暂无可用机场"

#### Scenario: 无可用航班

- **WHEN** API 返回某机场无直飞航班
- **THEN** 该机场在结果中显示"无直飞航班"

#### Scenario: 出发地输入

- **WHEN** 用户在出发地输入框输入城市名称
- **THEN** 系统通过高德行政区划搜索 API 返回城市名称联想列表
- **AND** 用户选择后填入城市名和对应机场代码

#### Scenario: 目的地选择

- **WHEN** 用户点击目的地下拉框
- **THEN** 系统展示全国省份列表
- **AND** 用户选择后填入省份名

#### Scenario: 日期选择

- **WHEN** 用户点击出发日期字段
- **THEN** 系统展示日期选择器
- **AND** 禁止选择今天之前的日期

### Requirement: 搜索结果可筛选

系统 SHALL 提供筛选开关，允许用户控制是否显示甩尾航班结果。

#### Scenario: 默认筛选状态

- **WHEN** 搜索结果首次加载
- **THEN** "甩尾航班"筛选开关默认开启
- **AND** 同时显示直飞和甩尾航班结果

#### Scenario: 关闭甩尾航班筛选

- **WHEN** 用户关闭"甩尾航班"筛选开关
- **THEN** 结果中仅显示直飞航班
- **AND** 甩尾航班区域隐藏

### Requirement: 搜索结果支持缓存

系统 SHALL 将航班搜索结果缓存到 `flight_search_cache` 表，TTL 为 4 小时。

#### Scenario: 缓存命中

- **WHEN** 用户在 4 小时内重复搜索相同的出发地、目的地、出发日期
- **THEN** 系统直接从缓存返回结果
- **AND** 不调用外部 API

#### Scenario: 缓存过期

- **WHEN** 缓存记录的 `expires_at` 小于当前时间
- **THEN** 系统丢弃该缓存
- **AND** 重新调用 Amadeus API 获取数据
- **AND** 更新缓存

### Requirement: 系统自动发现甩尾航班

系统 SHALL 对每条直飞搜索结果，异步扫描经停该目标城市的联程航班，若联程总价低于直飞最低价则标记为甩尾航班。

#### Scenario: 发现甩尾航班

- **WHEN** 直飞最低价为 ¥580（上海→长沙），系统扫描延程候选城市（如上海→长沙→昆明）
- **AND** 联程票价 ¥420 低于直飞最低价 ¥580
- **THEN** 该联程航班标记为甩尾航班
- **AND** 显示节省金额（¥160）和节省比例（27.6%）

#### Scenario: 联程不便宜

- **WHEN** 联程票价 ¥650 高于或等于直飞最低价 ¥580
- **THEN** 该联程航班不在甩尾列表中展示

#### Scenario: 甩尾航班异步加载

- **WHEN** 直飞搜索结果已展示
- **THEN** 甩尾航班区域显示加载态提示文字"正在寻找甩尾航班..."
- **AND** 甩尾结果返回后追加到对应城市卡片中
- **AND** 不影响直飞结果的展示

### Requirement: 甩尾航班显示风险提示

系统 SHALL 在每个甩尾航班旁展示醒目的风险提示。

#### Scenario: 甩尾航班附带提示

- **WHEN** 结果中包含甩尾航班
- **THEN** 每条甩尾航班旁显示风险提示图标和文字
- **AND** 提示内容包含"托运行李将到达终点站"和"违反航司运输条款"
- **AND** 提示内容包含"不可购买往返票（后半程将被取消）"

### Requirement: 机票搜索页面需要登录

系统 SHALL 保护 `/app/flights` 路径，未登录用户重定向到 `/sign-in`。

#### Scenario: 未登录访问机票搜索

- **WHEN** 未登录用户访问 `/app/flights`
- **THEN** 重定向到 `/sign-in`

#### Scenario: 已登录访问机票搜索

- **WHEN** 已登录用户访问 `/app/flights`
- **THEN** 页面返回 200 并渲染搜索表单

### Requirement: 搜索参数校验

系统 SHALL 在提交搜索前校验必填字段。

#### Scenario: 必填字段缺失

- **WHEN** 用户未选择出发地或未选择目的地或未选择日期就点击搜索
- **THEN** 缺失字段下方显示校验错误提示
- **AND** 不发起 API 请求

#### Scenario: 过去日期阻止

- **WHEN** 用户选择的出发日期是今天之前
- **THEN** 日期字段显示校验错误
- **AND** 搜索按钮不可点击
