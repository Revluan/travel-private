# Long-form docs

For things that don't fit in a spec but need to be written down:

- **ADRs** (Architecture Decision Records) — `adr/<NNN>-<title>.md`
- **Research notes** — `research/<topic>.md`
- **Runbooks** — `runbooks/<scenario>.md`
- **Design deep-dives** — `design/<topic>.md`

Specs describe **what** the system does. This directory describes **why** we chose to do it that way, **how** to operate it, and **what** we considered but rejected.

## ADR template

```markdown
# ADR-NNN: <title>

**Date:** YYYY-MM-DD
**Status:** Proposed | Accepted | Superseded by ADR-MMM | Deprecated

## Context
<why this decision is needed>

## Decision
<what we decided>

## Alternatives considered
<what else we looked at, why we rejected it>

## Consequences
<what this makes easier, what it makes harder>
```
